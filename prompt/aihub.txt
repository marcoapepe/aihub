we are going to create a React and FastAPI project called AIHUB.

The frontend has to be written in React and can use the existing code in available on frontend folder as reference.
But it is not mandatory.
If you have something better to use, go ahead. You can discard this code.

The frontend will have these features:
- User can upload PDF files
- User can see the PDF files on screen
- User can see all PDF files that were uploaded
- Every PDF file uploaded will have a serial number, starting with 1. An incremental counter. 
- All PDF files uploaded will be stored on files folder
- As the application will not have a database, we need to have a kind of database file where the #counter and pdf file name will be stored.

The Frontend will have a initial screen with two button, one to upload new PDF files and another button to see all PDF files that were uploaded

Once a document is uploaded, the screen will show the document on screen. 
User can scroll up and down on it. 

Important. Don't use the entire screen to show the document. 
On the right, we will have some extra features. 
Allocate 60% of the screen of the left to show the document. 
The other 40% on the right, we will use to one verry important function. 
We will have a text box where the user will input a prompt text. 
And a button that will call an API on the backend. 
And another text below to show the results from the API. 
Then below we will have another button that will save everything on the database file. 
This database file can be any stand alone database. You can choose the best way to do.
It will have:
- counter
- document name
- prompt that was inputted by user
- response from the API

The backend specifications:
The backend is one FASTAPI server that will have one endpoint only.
The endpoint will receive a prompt text and will call OpenAI API. 
We can choose GPT5 model for it initially. Later on we can a add a feature on screen to have more models to choose.
And the result of OpenAI API will be the response to the frontend.

The API KEY will be saved on a .env file.

You can start coding now.
